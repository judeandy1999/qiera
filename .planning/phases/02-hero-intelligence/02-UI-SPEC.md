**Backdrop:**
- `::backdrop` pseudo-element on the `<dialog>`: `background: var(--color-modal-backdrop)` (rgba(0,0,0,0.70))
- `prefers-reduced-motion`: no animation on open/close; content appears/disappears instantly. If a fade animation is added for non-reduced-motion users: opacity 0→1 over 200ms ease-out — apply via CSS `@media (prefers-reduced-motion: no-preference)` block only

**Modal container (the `<dialog>` element itself):**
- Max-width: 1120px **[ASSUMED — measure against card reference screenshots]**

## Accessibility Contract (HOME-09)

The following behaviors are mandatory for sign-off — verified by the executor during implementation, not deferred to Phase 10.

| Behavior | Implementation |
|----------|----------------|
| Keyboard open modal | "Learn more →" is a `<button>` (or `<a>` if routable — prefer `<button>` since it opens a modal, not navigates); activates on Enter and Space |
| Escape close | Native `<dialog>` cancel event — no extra listener |
| Focus trap | Native `showModal()` manages this |
| Focus restore | Native `<dialog>` returns focus to the triggering element on close |
| Tab cycle inside modal | All interactive elements inside the `<dialog>` are in the natural tab order; no `tabindex` manipulation needed |
| Carousel keyboard | Arrow left/right keys on carousel container, or tab to prev/next buttons and press Enter/Space |
| Carousel touch | CSS scroll-snap, native touch scroll — no JS library |
| Modal mobile full/near-full | At ≤767px: `width: 100vw; height: 100dvh; border-radius: 0; margin: 0` |
| prefers-reduced-motion | Wrap any modal open/close CSS animations in `@media (prefers-reduced-motion: no-preference)` — default is no animation at all |
| Section heading order | `<h1>` on hero (once, in Hero component), `<h2>` on section heading ("Growth Intelligence…"), no heading skipped |
| ARIA dialog label | `<dialog aria-labelledby="{modal-heading-id}">` — the modal H2 is the dialog label |
| Icon-only buttons | All icon-only buttons (carousel prev/next, modal close) have `aria-label` — no button is icon-only without a text alternative |
| Card icon alt | Card icon images: `alt=""` (decorative, title already provides the label) |
| Operating model image | Descriptive `alt` text (specified above in Operating Model Graphic section) |
