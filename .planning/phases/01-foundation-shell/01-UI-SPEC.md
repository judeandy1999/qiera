## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (CTA arrow gap), inline padding |
| sm | 8px | Compact element spacing (nav item internal padding, badge padding) |
| md | 16px | Default element spacing (nav item gaps, footer link gaps) |
| lg | 24px | Section padding, header vertical padding, mobile container gutter |
| xl | 32px | Layout gaps, footer column gaps |
| 2xl | 48px | Major section breaks, desktop container gutter |
| 3xl | 64px | Page-level spacing, header/footer max horizontal inset on large desktop |

Exceptions: **mobile menu touch targets minimum 44px** (nav links and the menu toggle/close buttons inside `MobileNav` must have a minimum hit area of 44×44px on touch viewports, even where visual padding alone would produce a smaller box — pad with invisible hit-area, not visible bulk, where the visual matches the reference at a smaller size).

Source: locked answer (orchestrator). Confirmed compatible with `01-RESEARCH.md` Pattern 1 (`@theme` spacing tokens).

### Header
- **Visual hierarchy / scan order:** The purple **Q** logomark is the first visual anchor; the primary CTA’s accent fill is the second; nav links are equal-weight text with no competing emphasis between them.
- Sticky: `position: sticky; top: 0; z-index: 40` from Phase 1 (reference doesn't conclusively show scroll behavior, but `01-RESEARCH.md` Pitfall 5 and `qiera-visual-fidelity.mdc`'s "Account for the sticky header using scroll-margin-top" rule both presuppose one exists; building it now avoids Phase 5 rework). **[ASSUMED — medium confidence, low-cost if wrong]**.
- Background: `--color-shell` (`#05060B`), full-bleed (no visible page background behind it)
- Height: content-driven, vertical padding `24px` (`lg`) top/bottom — matches the visual weight in `75976647-…png`/`top section.png`
- Content row: `Logo` (left) → desktop `<nav>` links (center-right, `16px`/`md` gaps) → primary CTA button (far right) — all within `--container-max`, horizontally centered
- At mobile widths, desktop `<nav>` + CTA collapse behind a single menu toggle button (`MobileNav`); Logo remains visible
- Nav link default color: `--color-text-primary` (all nav items render as equal-weight white/light-gray text in the reference — no visible "active route" state is shown in any Phase 1 file; if an active-state affordance is added, it must use `--color-accent`, never a second unrelated color)

### Footer
- Background: `--color-shell` (`#05060B`), full-bleed
- Row 1: `Logo` + tagline (left) → nav links (right), same v1 link set as Header, `16px`/`md` gaps — reads at `Label` size (14/400) rather than the Header's link size, matching the visibly smaller footer nav text in `Footer.png`
- Divider: `1px solid var(--color-border)` full-width rule between the nav row and the legal row
- Row 2 (below divider): copyright (`© 2026 QIERA. All rights reserved.`, `Label`/14/400, `--color-text-secondary`) left-aligned, legal links (Privacy Policy, Terms of Use, Cookie Policy) right-aligned, separated by a thin vertical divider glyph (`|`) or `16px` gap — reference shows a `|` separator; recreate as a styled `<span aria-hidden="true">` between `<li>` items, not literal text, so screen readers don't announce the pipe
- Vertical padding: `32px` (`xl`) top/bottom of the whole footer block, `24px` (`lg`) between the nav row and the divider

### Logo (`Q` mark + wordmark)
- Recreate the "Q" ring mark as an inline SVG component (RSC), fill `--color-accent`, per `01-RESEARCH.md` Pattern 3 — do not crop it from the reference PNG (violates the project's "no raster crops of reference screenshots as assets" rule)
- Wordmark "QIERA": `Heading`/20/600, `--color-text-primary`, immediately right of the mark
- Tagline "E-commerce Growth Intelligence": `Label`/14/400, `--color-text-secondary`, directly beneath the wordmark, tightly tracked (reference shows it visually smaller/tighter than standard nav text — treat as the same 14px token, do not introduce a 5th size)
- **[ASSUMED]** — whether the redrawn SVG is pixel-faithful enough is a judgment call per `01-RESEARCH.md` Assumption A2; the documented fallback (raster `public/images/qiera/logo.png` via `next/image`, per `FOUND-05`) absorbs this risk if the screenshot comparison shows a mismatch.

### Mobile navigation (`MobileNav`)
- Implementation: native `<dialog>` element (per `01-RESEARCH.md` Pattern 4) — zero new dependencies, gives focus trap / Escape-to-close / inert background / focus restore for free
- Panel background: `--color-secondary` (`#10121B`) — one step lighter than the page shell so the panel reads as an overlay, not a seamless extension of the background
- Open trigger: icon-only button, minimum 44×44px hit area (spacing exception), visible focus ring (`--color-focus-ring`)
- Close: dedicated close button (also 44×44px minimum) **and** native `Escape` (via the dialog's `cancel`/`close` events — no manual `keydown` listener, per Don't-Hand-Roll)
- Nav links inside the panel: stacked vertically, `Label`/14/400 (or bumped to `Body`/16/400 for easier touch legibility — implementer's discretion within the declared type scale; do not introduce a new size)
- Motion: if any open/close transition is added (e.g. panel slide/fade), it must respect `prefers-reduced-motion` — reduce to an instant/opacity-only transition when the media query matches (forward-looking; native `<dialog>` has no required animation, so the simplest compliant default is **no animation at all** in Phase 1, deferring transition polish to a later pass)
